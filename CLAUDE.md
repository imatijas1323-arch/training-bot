# Swimming Training Bot — Справочник проекта

## Суть проекта
Telegram-бот для управления тренировками по плаванию. Ученики регистрируются, смотрят расписание на неделю и бронируют тренировки (в бассейне или удалённо). Вся база данных — Google Таблица.

## Файлы
| Файл | Роль |
|---|---|
| `main.py` | Основной бот — все хэндлеры aiogram, кэш, уведомления, фоновые задачи |
| `sync_bd.py` | Standalone-скрипт синхронизации (дублирует логику из main.py, запускается вручную) |
| `test_sheets.py` | Проверка подключения к Google Sheets |
| `credentials.json` | Ключ сервисного аккаунта Google (swim-bot-account@calener-495720.iam.gserviceaccount.com) |
| `logo.png` | Логотип бота — показывается при /start и в главном меню |

## Стек
- **Python** + **aiogram 3.x** — Telegram бот (polling режим)
- **gspread** + **google-auth** — Google Sheets API
- **БД нет** — Google Таблица является единственным хранилищем данных

## Поток работы бота
```
/start → фото с лого + кнопка "🚀 Начать пользоваться"
  ├─ не зарегистрирован → выбор имени → подтверждение → сохранение Telegram ID
  └─ зарегистрирован → Главное меню (фото + inline-кнопки)
        ├─ 📅 Расписание → список дней недели → карточка дня → забронировать / отменить
        ├─ 📝 Мои записи → все забронированные тренировки
        ├─ 🎫 Абонемент → остаток тренировок
        └─ 👤 Профиль → метры за неделю / год / сравнение с прошлым годом
```

## Кэш (in-memory)
```python
_user_cache           # telegram_id → имя (живёт до перезапуска)
_tid_cache            # имя → telegram_id
_bd_rows              # данные листа BD (TTL 5 минут)
_last_booking         # имя → дата последнего бронирования "DD.MM.YYYY" (персист. в строке 7)
_inactivity_notified  # set имён кому уже отправлено "давно не виделись"
_week_session_notified  # флаг: уведомляли ли о расписании в этой сессии
_reminded_training    # set "имя|дата" — кому уже отправлено напоминание за 2ч
_reminder_date        # текущая дата для сброса _reminded_training в полночь
_known_plans          # "имя|дата" → текст плана (для детектирования нового плана)
_state_notified       # set "имя|дата" — кому уже отправлен запрос о состоянии
_state_date           # дата для сброса _state_notified в полночь
_multi_select         # telegram_id → set дат (мультибронирование)
```
- `_ensure_bd()` — обновляет кэш BD если TTL истёк
- `_invalidate_bd()` — принудительный сброс кэша (вызывается после бронирования)
- `load_all_tids()` — загружает все Telegram ID из строки 6 таблицы
- `load_last_bookings()` — загружает последние даты бронирований из строки 7
- `save_last_booking(name, date)` — сохраняет дату в строку 7 (персистентность)

## Система уведомлений

### Тренеру (TRAINER_ID = 80639782) — мгновенно:
- `🏊 {имя} забронировал тренировку — {дата}`
- `🏠 {имя} записался удалённо — {дата}`
- `❌ {имя} отменил запись — {дата}`
- `🆕 {имя} зарегистрировался` — при первой регистрации (имя TG, username, ID)
- `❓ Новый пользователь хочет зарегистрироваться` — кнопка «Меня нет в списке»
- `📊 {имя} оценил тренировку {дата}: {состояние}` — после state_checker

### Пользователям:
| Событие | Когда | Текст |
|---|---|---|
| Расписание готово | Тренер ставит "текущая неделя" (через ~15 сек) | `📅 Расписание на {неделя} готово!` |
| Абонемент заканчивается | После бронирования, остаток ≤ 2 | `🟡 Осталось {N} тренировки` |
| Абонемент = 0 | После бронирования | `🔴 Абонемент закончился` |
| Абонемент < 0 | После бронирования | `🚨 Вы ушли в минус: {N} тренировок` |
| Еженедельный итог | Воскресенье 20:00 | `📊 Тренировок: N, Метров: X м` (только если были тренировки) |
| Давно не виделись | Ежедневно 10:00, если 14+ дней без записи | `👋 Давно не виделись — {N} дней без тренировки` |
| Напоминание о тренировке | За ~2 часа до начала | `⏰ Через ~2 часа тренировка в {время}` |
| Новый план | Каждые 5 мин, если тренер добавил план | `📋 Тренер написал план! {день} {дата} — {план}` |
| Оценка состояния | Через ~2ч после тренировки | inline-кнопки с 7 вариантами (шкала 1–10) |

## Смена недели (механика через панель тренера)
1. Тренер нажимает «⏹ Закрыть неделю» → бот пишет `прошедшая неделя` в ячейку + сворачивает строки через Sheets API
2. Тренер нажимает «📂 Развернуть следующую неделю» → строки разворачиваются, показывается предпросмотр 7 дней (статус "не выбрано")
3. В режиме предпросмотра тренер может тапать дни и ставить время (14:00, 20:00, своё, нет тренировки)
4. «✅ Открыть эту неделю» → пишет `текущая неделя` в ячейку, ученики НЕ уведомляются автоматически
5. «📢 Уведомить учеников о расписании» → тренер вручную отправляет уведомление всем
6. `week_watcher` больше не шлёт уведомления сам — `_week_session_notified = True` ставится при открытии

**Ноу-хау:**
- Маркер недели — это текст в дропдауне (`текущая неделя` / `прошедшая неделя` / `не выбрано`). Бот пишет напрямую через `src.update()`, это эквивалентно выбору из дропдауна если текст совпадает точно
- Сворачивание строк — через Sheets API v4 `batchUpdate` → `updateDimensionGroup` с `collapsed: true/false`
- Группы строк листа 2026: depth=1 (год), depth=2 (квартал), depth=3 (неделя, по 35 строк)
- `toggle_week_collapse()` берёт группу с наименьшим диапазоном (depth=3), чтобы не задеть depth=1/2
- `tr_find_next_week_row()` ищет следующую неделю через `get_sheet_row_groups()` по startIndex (depth=3) — надёжнее чем сканировать строки, т.к. scan мог вернуть вторник вместо понедельника
- `load_week_rows()` при старте читает таблицу и восстанавливает `_week_marker_row`, `_prev_week_row`, `_week_collapsed` — всё сбрасывается при перезапуске без этого
- `_prev_week_row` = строка с максимальным индексом среди всех `прошедшая неделя` (исторически их много)
- `_week_collapsed` синхронизируется с реальным состоянием collapse из Sheets API при старте

**Кнопки «Свернуть / Развернуть» (панель тренера):**
- «📦 Свернуть» — только сворачивает строки в таблице, статус не меняет; в боте 7 дней прячутся, видна кнопка «📂 Развернуть текущую неделю»
- «📂 Развернуть текущую неделю» — разворачивает строки, возвращает 7 дней
- «📂 Развернуть прошлую неделю» — появляется когда нет активной недели; показывает дни прошлой недели в режиме только чтения (тап → попап со временем)
- Свернуть/развернуть не меняют статус → `week_watcher` не видит изменений → уведомления не летят

**plan_checker — ноу-хау:**
- Уведомляет только когда план ИЗМЕНИЛСЯ относительно уже известного значения
- Новый ключ (смена недели) = просто записывается в baseline без уведомления
- Было: `old_plan = _known_plans.get(key, "")` → при новом ключе old_plan="" → любой план = "новый" → спам
- Стало: `if key not in _known_plans: _known_plans[key] = new_plan; continue`

При статусе "не выбрано": BD очищается, бот показывает "⏳ Расписание ещё не готово"
