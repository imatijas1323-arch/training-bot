### Детали расписания (мультивыбор)

```
Состояния ячейки в _multi_select[uid][date]:
  "pending"  → день выбран, тип не выбран → показан ◉
  "pool"     → выбрана очная тренировка  → показан ☑️
  "remote"   → выбрана удалённая         → показан ☑️

Флоу мультивыбора:
  toggle_{date} ──► если remote-only день → сразу "remote"
                    иначе показать [🏊 С тренером] [🏠 Удалённо]
  set_pool_{date} ──► _multi_select[uid][date] = "pool"
  set_remote_{date} ──► _multi_select[uid][date] = "remote"
  book_selected ──► для каждой выбранной даты:
                    set_booking(name, date, True)
                    if remote: set_remote_booking_comment(name, date)
```

### 5.1 Расписание (tr_schedule)

```
Три режима:

[Нормальный — неделя открыта]
  Обзор 7 дней: Пн 🟢 09:00 / Вт 🔴 нет тренировки / ...
  Тап по дню → [14:00][20:00][Другое][Нет тренировки]
               → подтверждение → tr_set_time() → 2026!B
  [⏹ Закрыть неделю] → маркер "прошедшая неделя" + collapse строк
  [📦 Свернуть] / [📂 Развернуть текущую]

[Предпросмотр — следующая неделя развёрнута]
  [📂 Развернуть следующую неделю] → tr_find_next_week_row()
  Редактируем времена по дням
  [✅ Открыть эту неделю] → маркер "текущая неделя"
  [📢 Уведомить учеников] → рассылка о расписании

[Предыдущая неделя]
  Read-only просмотр прошлой недели
```

**Добавление ученика:**
```
[➕ Добавить] → Ввод имени → Подтверждение
  → add_new_student_to_sheets(name):
    1. insertDimension в 2026 и абонемент (или reuse orphan)
    2. setDataValidation (чекбоксы) для ~377 строк
    3. Адаптация формул из предыдущей колонки (regex prev→new)
    4. Запись имени (row1, row12), TG ID пусто, метры=0
    5. Абонемент: формула SUM, остаток =O4
    6. USER_COLUMNS + Meta (["student", name, col])
    7. GradeCurrent.append_row([name, "", ""])
```

**Удаление ученика:**
```
В архиве → [🗑 Удалить навсегда] → Подтверждение
  → delete_student(name):
    1. deleteDimension в 2026 и абонемент
    2. Удаление строк из Meta (student + archived)
    3. Удаление строки из GradeCurrent
    4. Очистка всех кэшей (USER_COLUMNS, _archived_students,
       _known_grades, _tid_cache, _user_cache, _last_booking)
```

## 6. Фоновые задачи и уведомления

```
asyncio event loop
│
├── week_watcher()          ─ каждые 15 сек
│     Следит за маркером "текущая неделя" в 2026
│     При смене: инвалидирует BD кэш
│     При открытии: уведомляет всех учеников
│
├── grade_checker()         ─ каждые 5 мин
│     Читает GradeCurrent, сравнивает с _known_grades
│     При изменении: notify_user + запись в Grades
│
├── plan_checker()          ─ каждые 5 мин
│     Читает план из BD для каждого записанного ученика
│     При изменении: notify_user("📋 Тренер написал план!")
│     Новые ключи (смена недели): только baseline, без уведомления
│
├── training_reminder()     ─ каждую минуту
│     Для каждой брони сегодня: вычислить время до начала
│     Если 110–130 мин: notify_user("⏰ Через ~2 часа...")
│     Отслеживает _reminded_training, сброс в полночь
│
├── state_checker()         ─ каждую минуту
│     Для завершённых тренировок (2+ ч назад):
│     Отправить запрос оценки состояния (7 кнопок)
│     Ответ записывается в 2026!+4 строка
│     Отслеживает _state_notified, сброс в полночь
│
├── inactivity_checker()    ─ ежедневно в 10:00 МСК
│     Если дней без брони >= 14: notify_user("👋 Давно не виделись")
│     Разово на ученика (до следующего бронирования)
│
└── weekly_report()         ─ воскресенье 20:00 МСК
      Подсчёт тренировок и метров за неделю
      notify_user("📊 Итог недели: N тренировок, X м")
      Только если были тренировки
```

### Матрица уведомлений

```
Событие                       │ Кому          │ Когда
──────────────────────────────┼───────────────┼────────────────────
Расписание открыто            │ Все ученики   │ ~15 сек (watcher)
Новый план тренера            │ Записанные    │ ≤5 мин (checker)
Напоминание о тренировке      │ Записанный    │ за 110–130 мин
Запрос оценки состояния       │ Записанный    │ через 115–140 мин
Новый грейд                   │ Ученик        │ ≤5 мин (checker)
Давно не виделись             │ Неактивный    │ ежедн. 10:00
Итог недели                   │ Тренировавшиеся│ вс 20:00
Абонемент ≤ 2                 │ Ученик        │ после бронирования
Абонемент = 0 или < 0         │ Ученик        │ после бронирования
Новый ученик зарегистрировался│ Тренер        │ сразу
Новое бронирование            │ Тренер        │ сразу
Отмена бронирования           │ Тренер        │ сразу
Оценка состояния получена     │ Тренер        │ сразу
Нет в списке (незнакомый)     │ Тренер        │ сразу
Пополнение абонемента         │ Ученик        │ сразу
```

## 7. In-memory состояние

```python
# Идентификация
_user_cache:          dict[int, str]    # telegram_id → имя
_tid_cache:           dict[str, int]    # имя → telegram_id
_authenticated_trainers: set[int]       # авторизованные тренеры

# Кэш расписания (TTL 5 мин)
_bd_rows:             list              # строки листа BD
_bd_ts:               float             # timestamp последнего sync

# Управление неделей
_week_marker_row:     int               # строка с "текущая неделя"
```

## 8. Управление учениками

### Жизненный цикл ученика

```
                    [Добавить ученика]
                           │
                           ▼
              add_new_student_to_sheets()
                           │
                     ┌─────┴─────┐
                     │  Активный │  ← основной список
                     └─────┬─────┘
                           │ [В архив]
                           ▼
                    archive_student()
                    Meta: ["archived", name]
                           │
                     ┌─────┴─────┐
                     │  В архиве │
                     └──┬──────┬─┘
                        │      │ [Вернуть]
                        │      ▼
                        │  restore_student()
                        │  удалить строку из Meta
                        │
                        │ [Удалить навсегда]
                        ▼
                  delete_student()
                  deleteDimension 2026 + абонемент
                  удалить из Meta + GradeCurrent
                  очистить все кэши
```

## 9. API-точки для Mini App

> При разработке Mini App / Web App потребуется бэкенд API.  
> Ниже — список операций, которые нужно перенести из бота.

### Чтение данных

| Операция | Источник | Функция в боте |
|---|---|---|
| Расписание текущей недели | BD sheet | `_ensure_bd()` |
| Статус бронирования ученика | BD sheet | `_bd_rows` |
| Остаток абонемента | абонемент!row5 | `tr_get_subscription()` |
| Метры за неделю / год | 2026!row5/row10 | `cb_profile()` |
| Текущий грейд | GradeCurrent | `_known_grades` |
| Список участников дня | BD sheet | - |
| Детали тренировки (план/объём) | BD sheet | `cb_day_detail()` |

### Запись данных

| Операция | Цель | Функция в боте |
|---|---|---|
| Забронировать тренировку | 2026!чекбокс | `set_booking()` |
| Отменить бронирование | 2026!чекбокс | `cancel_booking()` |
| Указать удалённую тренировку | 2026!комментарий | `set_remote_booking_comment()` |
| Оценить состояние | 2026!+4 строка | `set_state()` |
| Пополнить абонемент | абонемент лог | `tr_add_subscription()` |
| Написать план | 2026!план строка | `tr_write_plan_for_user()` |
| Написать объём | 2026!объём строка | `tr_write_vol_for_user()` |
| Сменить грейд | GradeCurrent | `tr_set_grade()` |

### Рекомендации для Mini App архитектуры

```
┌─────────────────────────────────────────────────────────────┐
│                    РЕКОМЕНДОВАННАЯ СХЕМА                    │
│                                                             │
│  Telegram Mini App (фронтенд)                               │
│  │                                                          │
│  ▼                                                          │
│  REST API / FastAPI                                         │
│  ├─ GET  /schedule          → возвращает BD sheet           │
│  ├─ POST /booking           → set_booking()                 │
│  ├─ POST /booking/cancel    → cancel_booking()              │
│  ├─ GET  /profile/{name}    → метры, абонемент, грейд       │
│  ├─ GET  /subscription/{name}                               │
│  ├─ POST /state             → set_state()                   │
│  └─ (тренер) /trainer/*                                     │
│  │                                                          │
│  ▼                                                          │
│  Google Sheets (через gspread)                              │
│  Тот же spreadsheet — бот и Mini App работают с одной БД    │
└─────────────────────────────────────────────────────────────┘

Ключевые решения:
  • Аутентификация: telegram initData (WebApp.initData) →
    валидируется через HMAC-SHA256 с BOT_TOKEN
  • Кэш BD: TTL 5 мин, инвалидация после записи — можно
    сохранить в Redis или in-memory
  • Уведомления: бот Telegram остаётся для push-уведомлений
  • Тренер vs ученик: роль определяется по telegram_id
    (TRAINER_ID из .env)
```
